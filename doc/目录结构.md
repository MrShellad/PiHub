src/
├── domain/            # 核心领域层 (没有任何外部依赖，如 tokio/webrtc)
│   ├── models.rs      # 领域实体与值对象 (如 InviteCode, SessionState)
│   ├── ports.rs       # 抽象端口 (定义 P2pEngine 和 IpcEmitter 的 Trait)
│   └── mod.rs
├── application/       # 应用服务层 (组合领域逻辑，编排业务流)
│   ├── use_cases.rs   # 核心业务用例 (如 CreateRoom, JoinRoom)
│   └── mod.rs
├── infrastructure/    # 基础设施层 (实现 ports 中的 Trait，处理脏活累活)
│   ├── webrtc_adapter.rs # WebRTC 的具体实现
│   ├── stdio_adapter.rs  # 标准输出的具体实现
│   └── mod.rs
├── presentation/      # 表现/接口层 (接收外部输入)
│   ├── ipc_listener.rs   # 监听 Stdin 循环，将 JSON 路由到 application
│   └── mod.rs
└── main.rs            # 组装层 (依赖注入，把一切装配起来)


在 Rust 中落地 DDD，核心思想是：依赖倒置（Dependency Inversion）。领域层和应用层只定义 Trait（接口/端口），基础设施层去实现这些 Trait（适配器）。
